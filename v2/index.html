掃碼看履歷

Robin - 吳孟蘋

您好，我是孟蘋，感謝您撥空閱讀我的履歷，請容我先誠懇的說明轉職PM的想法及目標，再介紹我的工作經歷。

對轉職PM的想法並非一時，曾配合過沒有程式開發經驗的PM或Sales，造成許多需求較複雜的案件進行到一半出問題，比較常見的是『誤會需求是曾經做過的功能』，其次是『因著急又找錯人，造成同事之間的不悅與不信任』，因此我希望PM是能有基礎程式能力的背景。

雖然我沒有實際管理的經驗，但是我一直需要與PM、UIUX設計師和後端工程師有密切交流、保持友好，而在我的工作經驗中，還沒有過和這三方交惡的事情發生，恰當處理人際關係是我認為可以嘗試PM的第一個理由。

第二理由是，工作態度謹慎、穩定，這是直至目前從同事、長官聽到對我的多數評價，有時工作流程上前段人員被拖延導致跟著被延後，也不會不負責任的就下班離去；如果遇到不可預期的狀況，也會先想出替代方案解決現下問題。

第三理由是，不會有所謂的“帶著前一份工作的壞習慣”，前端的經驗裡，也多次需要統整三方工作時程上報給PM，也會與使用者溝通問題、與老闆確認功能使用狀況，我認為在PM職務上我所欠缺的時程安排經驗、規格文件撰寫是可以透過實際任職並依公司內部作業文化後被訓練，我相信能上手時間會較短且穩定度較高。

第四理由是，不太唱反調，但會看時機提出看法，習慣先滿足主管所需、了解目前狀況，取得雙方信任後才提出；但是不限於面對主管，我過去配合的設計師或工程師，儘管我略懂他們的工作內容，在溝通時，只要涉及對方的專業領域，總是先聽他們想如何發揮，再來加入我自己的想法，多次這種態度配合後，我發現這能讓他們更樂於在討論階段時提出自己的意見。

此外，對於自己轉職所需要面臨的問題相當清楚，例如會有一段相比資深前端較劣勢的薪資期間，又例如，先前因在接案公司或通路的電商網頁，較無單一產業的背景經驗會較弱勢，但是我希望被錄取後可以盡快追上這段缺口，也許透過唸書或實際上工，期許能夠成為能夠獨立主導大型專案的PM。

以上是我認為可以錄取我的參考重點，而為了擁有更多即戰力，利用私人時間練習Functional Map、Flow Chart、UI Flow、Wireframe...等，並在下方工作經歷後以模擬專案的方式呈現。

Introduction｜簡介

寶寶託付日記-發想來自拜訪了保母朋友後，得知有此種聯絡簿，並成為了我模擬專案的主題，它會是一款針對托嬰製作的工具APP，透過電子化，方便保母工作紀錄並協助家長了解寶寶。

現行紙本聯絡簿參考：

紙本聯絡簿1
紙本聯絡簿2

本APP將有兩種使用者：
（一）主要目標使用者：從事照顧寶寶之工作者（以下稱保母）
（二）次要目標使用者：委外托嬰之家長（以下稱家長）

除了將表格電子化，還需要思考什麼特色可以使APP被廣泛使用，初步評估的功能有：

  • 主要功能
    1. 註冊/登入

      1.使用者身份
      2.忘記密碼
      3.驗證手機號碼

    2. 聯絡簿紀錄與回覆(多項表格填寫、圖片上傳)

      1.表格欄位填寫
      2.圖片上傳(需在前段先被壓縮後，才存進資料庫)
      3.發送簽名提醒至連結的家長帳號
      4.聯絡簿更改權限

    3. 寶寶管理

      1.基本資料表格欄位填寫(含圖片上傳)
      2.關聯表單填寫(6張)
      3.寶寶列表管理
      4.發送連結邀請家長帳戶

    4. 使用者管理

      1.基本資料建置(表格欄位與上傳圖片)
      2.APP推播開啟管理

  • 次要功能
    1. 保母任務快捷按鈕

      1.首頁任務快捷鍵設定(保母使用者限定)

    2. 個人化提醒

      1.提醒事件設定(項目與時間設定)

    3. 數據紀錄統整與分析

      1.統計依被記錄的任務(喝水,喝奶,排便,活動參與度)

    4. 匯出紀錄pdf檔

      1.可選期間匯出

    5. 保母使用者基本資料

      1.上傳保母證照

【關於我】

我是孟蘋，來自古都色彩濃厚的台南，休閒時間喜歡看美劇老片、抽空閱讀CNN報導文章，依自己的步調來讀學英文，現階段兩年內多益成績為745分；除了工作與學習，壁球是我相當喜歡的運動，有時也會參加公司羽球社團；本生性格較慢熱，工作時偏向符合主管期待並在熟悉後會提供自己的想法，同時也尊重視覺設計者或後端工程師的個別專業，同事間合作也都相當順暢。

【經歷】

求學時期經歷過不同領域，高職唸土木營建系，升大學時在勤益科大景觀設計系所，後來因南部家裡需要幫忙，一年後轉回台南，畢業於崑山科技大學房地產開發與管理系，後來報名了網頁設計師的課程，雖然又是一個新的環境，但從寫網頁到與不同人員交涉，多元、具挑戰性的工作內容中似乎能持續獲得成就感。

網頁前端處在PM、設計、後端工程之間，在這個位置常常會遇到需要做工作項目分派與溝通的角色，和設計師討論版面呈現、和後端討論時間夠不夠、有沒有折衷快速且簡單一點的方式，而對PM就是需要幫他們第一時間過濾問題，雖然涉及範圍大，但喜歡看到各個角色都能各司其職的發揮，也以能案子能滿意且快速的結案為樂。


在工作四年後，離職到北部前有幾個月想休息，由於不想留白，順勢報名了四個月的課(課程內容：程式設計 asp.net , C# / 系統分析)，以休息的心態多看看南部以外的世界。


【結語&未來展望】

在網頁設計領域打滾多年，我認為我幫助自己在的職場中能順利完成任務的主要原因是規劃與溝通的能力，並非我有一口舌粲蓮花或是前端技術特別高超，而是我喜歡順道了解一個指令或一個抱怨的背後原因，常常因為有去了解，與對方認知達成一致才動工，尤其在對接後端工程師上，因本身前端的工作對資料庫等有基本的認識，能幫助彼此簡化工作量或是清楚欄位需要設定甚麼，多次的配合下自然能與共事對象培養出的工作默契。

在休息的幾個月中接觸的課程中有系統分析這一門，腦海中更加融合了第一份工作內容的體驗，後來剛進燦坤時，第一次收到寫了簡易流程規劃的提案，久了越看越符合偏好的職涯方向，不過雖然對網站架構、頁面之間的連結性、互動性有深刻的了解，但所有工作經驗仍是在終端位置居多，以初期角度開始的工作經驗還算一張白紙，但我願意接受跨職的挑戰。
